# 共享规则

所有模式通用的规则，避免在各文件中重复。

## 数据完整性

所有原始数据必须逐字保留，不可改写。数字、百分比、名称、专有名词保持原样。

## 输出规范

- **语言一致**：`language=zh` → 中文输出；`language=en` → 英文输出
- **禁止颜色代码**：不使用 `#RRGGBB` / `rgb()` / `hsl()`，使用颜色描述词
- **禁止尺寸描述**：不使用像素级尺寸（`pt`、`px`、`像素`）和比例尺寸（`占宽 XX%`、`X×X 网格`），改用相对大小词；宽高比通过调用参数传递，不在提示词文本中描述
- **禁止代码围栏**：最终输出不包含 ` ``` ` 代码围栏
- **禁止元评论**：不输出"这是提示词"等说明文字

## 提示词写作规则

### 视觉精度

始终描述背景纹理（如 off-white aged paper, black halftone shadows）和字体风格（如 handwritten, serif print, colorful block-lettering）。省略这些会导致图像模型做出破坏预期美感的任意选择。

### 语义对应

每个图标、插图或装饰元素必须与相邻文本内容语义对应，避免使用可适用于任何主题的通用装饰元素。

### 标点卫生

描述风格、布局结构、颜色、纹理、情绪时，不使用引号。引号仅用于需要在图像中呈现为字面文本的内容（如标题文案、标签文字），用双引号 `"..."` 包裹。

## 提示词长度

- 目标：500-800 英文词（约 2500-4000 字符）
- 超过 5000 字符需精简

## 精简原则

1. 移除像素级尺寸描述（如 `60px wide`、`40px tall`），改用相对大小词（"large"、"medium"）
2. 合并相似元素描述，避免逐个展开
3. 使用自然语言段落式描述，而非逐项规格列表
