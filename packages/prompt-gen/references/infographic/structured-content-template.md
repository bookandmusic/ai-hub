# 结构化内容模板

## 模板结构

按以下三个阶段生成设计就绪的内容：

### 阶段 1：高层大纲 (High-Level Outline)

```markdown
# [信息图标题]

## 概述
[简短概述，1-2 句话]

## 核心要点
1. [要点 1 标题]
2. [要点 2 标题]
3. [要点 3 标题]
...

## 数据概览
- 主要数据点：[数据列表]
- 关键对比：[对比项]
- 趋势/变化：[趋势描述]
```

### 阶段 2：内容开发 (Section Development)

```markdown
## [要点 1 标题]

### 核心内容
[详细内容描述]

### 数据支持
- 数据 1: [数值 + 单位]
- 数据 2: [数值 + 单位]

### 视觉建议
- 图表类型：[建议的图表类型]
- 视觉元素：[建议的视觉元素]
- 颜色建议：[建议的颜色]

---

## [要点 2 标题]

### 核心内容
[详细内容描述]

### 数据支持
- 数据 1: [数值 + 单位]
- 数据 2: [数值 + 单位]

### 视觉建议
- 图表类型：[建议的图表类型]
- 视觉元素：[建议的视觉元素]
- 颜色建议：[建议的颜色]
```

### 阶段 3：数据完整性检查 (Data Integrity Check)

```markdown
## 数据完整性检查

- [ ] 所有原始数据已包含
- [ ] 数字/名称逐字保留
- [ ] 单位正确
- [ ] 逻辑关系正确
- [ ] 无遗漏关键信息
- [ ] 无添加未授权信息
```

## 使用规则

1. **数据完整性**：所有原始数据必须逐字保留，不可改写
2. **逻辑清晰**：内容组织符合信息图逻辑
3. **视觉就绪**：每个要点附带视觉表达建议
4. **层级分明**：标题 - 子标题 - 内容的层级清晰
5. **无授权添加**：不添加用户未提供的信息
6. **精简原则**：视觉建议中避免像素级尺寸描述，使用相对大小词（如"large"、"medium"）
